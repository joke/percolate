= Compile-time switches
:navtitle: Compile-time switches

Percolate takes a handful of annotation-processor options (`-A<name>=<value>`), each affecting either the
generated *text* or a debug side-channel — never the mapping behaviour itself. This page shows every option
with a real before/after of its generated effect.

[cols="2,1,4"]
|===
| Option | Default | Effect

| `percolate.docTags`
| `false`
| Wraps each generated method body in `// tag::<method>[]` / `// end::<method>[]` comments (used to build
  this very manual).

| `percolate.locals.final`
| `false`
| Prefixes every hoisted local variable declaration with `final`.

| `percolate.locals.var`
| `false`
| Declares every hoisted local with `var` instead of its explicit type.

| `percolate.parameters.final`
| `false`
| Declares every generated method's parameters `final`.

| `percolate.methods.final`
| `false`
| Declares every generated `@Override` method `final`.

| `percolate.classes.final`
| `false`
| Declares the generated `<Mapper>Impl` class `final`. **Changed default**: previously the generated class
  was unconditionally `final`; it is now `final` only when this option is set.

| `percolate.nullable.annotations`
| _(none)_
| A comma-separated list of additional `@Nullable` annotation FQNs to recognise, alongside JSpecify's own.

| `percolate.debug.graphs`
| `false`
| Writes one GraphViz `.dot` file per generated method (and per container-element scope) to the
  annotation-processing output directory.

| `percolate.time.zone`
| _(none)_
| A project-wide default `ZoneId` for xref:temporal-mapping.adoc[temporal mapping]'s zone bridge, used
  whenever a binding declares no `@Map(zone = …)` of its own.
|===
