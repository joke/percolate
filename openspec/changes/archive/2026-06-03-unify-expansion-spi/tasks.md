(Discovered via percolate-integration's two-method `PersonMapper`: the new `SourceDescentExpander` routes the param root through `SlotResolver.isParameterRootSlot`, which keyed off `currentMethod`; the deleted `PathSegmentExpander` SAT'd off the typed slot directly. Fix: `SourceParams.forSlot(node)` reads `MethodScope.getMethod()`; same change in `RealisationDiagnosticsStage.isParameterRootFailingSlot`. See [[project_param_root_uses_node_scope]].)
- [x] 3.9 Deduplicate structurally-identical emitted steps in `FrontierMatcher.produce` (signature: strategy FQN | intent | scope | output | input types). `CombinatorialMatch` offers each in-view candidate to `bridge(from,to)`, but a container's target-driven wrap/unwrap-by-synthesis branch ignores `from`, so N candidates emitted N identical container steps → N equal-cost twin sub-groups. (Discovered via the `List<Optional<Address>> → Optional<Set<Address>>` chain forking into two identical `Optional<Person.Address>` element nodes. See [[project_container_step_dedup]].)
