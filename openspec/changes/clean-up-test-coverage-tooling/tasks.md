## 6. Raise spi to the shared threshold, diagnosing variance rather than tolerating it

Baseline recorded (single run, history not yet cleared/repeated): **70% line coverage, 13% mutation kill, 19% test strength** (need 95/85/90). This is well below and consistent with the range `engine-test-quality` documented before `dissolve-private-type-universe` (16-26% killed, 24-40% strength, 70-73% coverage) — meaning the gap is a genuine, large test-coverage shortfall, not measurement noise from the removed javac fixture. Closing it is real test-writing work across `spi`'s public surface, not a quick fix.

**Resolved — root cause was not a test-coverage gap.** 3 cleared-history runs (`rm -f spi/build/pitHistory.txt && ./gradlew :spi:pitest --no-configuration-cache --rerun`) showed line coverage stable (~96-97%) but mutation kill/test strength unstable and far below floor (47%/48% → 54%/55% → 59%/60%), matching the `reactor-blocking` (task 8.3) symptom shape. Inspecting a specific "survived" mutant (`ResolveCtx.arrayComponent`'s `isArray` guard) that an existing `ResolveCtxSpec` assertion should logically kill led to running `ContainerSpec` directly (`./gradlew :spi:test --tests ContainerSpec`), which **failed outright** (not via pitest) with `IllegalArgumentException: Comparison method violates its general contract!` from `org.spockframework.runtime.SpecRunHistory.sortFeatures` / `OptimizeRunOrderExtension`. Spock's `optimizeRunOrder` (on by default) persists per-spec run-history to a per-user-home file (`~/.spock/RunHistory/<SpecName>`) shared across **every concurrent JVM on the machine** — under pitest's own minion-level parallelism, concurrent writes race and corrupt that file, intermittently crashing whole spec classes during test discovery, which pitest then misreports as widespread survived mutants. `processor/src/test/resources/SpockConfig.groovy` already disables this (`runner { optimizeRunOrder false }`, added in the `type-query-seam` change) but was never propagated to `spi`/`strategies-builtin`/`reactor`/`reactor-blocking` when pitest was newly enrolled in them by this change. Added `spi/src/test/resources/SpockConfig.groovy` with the same `runner` block (no `mockMaker` override — no spec in `spi` spies a final class). Re-verified 3 cleared-history runs, all stable at **97% line coverage / 96% mutation kill / 97% test strength** — no new tests needed. Full details in `pitest-attribution-repro.md` in this change directory (superseded by this resolution, kept as investigation log).
