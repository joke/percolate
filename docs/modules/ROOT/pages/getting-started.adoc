= Getting started
:navtitle: Getting started

This page adds percolate to a Maven or Gradle build and walks through a first mapper.

== How percolate is packaged

Percolate is a **compile-time-only** tool. You need two things on the build:

* the **annotations** (`annotations`) on the compile classpath â€” `@Mapper` and `@Map`; and
* the **annotation processor** (the `percolate` starter, which bundles the engine and the built-in
  strategies) on the annotation-processor path.

The generated mapper has **no runtime dependency** on percolate. Versions of percolate's own modules are
managed by the `bom` platform, so you pin the version once.

== Gradle

[source,groovy,subs="+attributes"]
----
dependencies {
    annotationProcessor platform('io.github.joke.percolate:bom:{percolate-version}')
    annotationProcessor 'io.github.joke.percolate:percolate'

    compileOnly platform('io.github.joke.percolate:bom:{percolate-version}')
    compileOnly 'io.github.joke.percolate:annotations'

    // optional: reactive (Flux/Mono) container support
    // annotationProcessor 'io.github.joke.percolate:reactor'
}
----

== Maven

The BOM is imported under `dependencyManagement`; the processor is supplied through the compiler plugin's
`annotationProcessorPaths` (which does not read the BOM, so it carries an explicit version).

[source,xml,subs="+attributes"]
----
<dependencyManagement>
  <dependencies>
    <dependency>
      <groupId>io.github.joke.percolate</groupId>
      <artifactId>bom</artifactId>
      <version>{percolate-version}</version>
      <type>pom</type>
      <scope>import</scope>
    </dependency>
  </dependencies>
</dependencyManagement>

<dependencies>
  <dependency>
    <groupId>io.github.joke.percolate</groupId>
    <artifactId>annotations</artifactId>
    <scope>provided</scope>
  </dependency>
</dependencies>

<build>
  <plugins>
    <plugin>
      <groupId>org.apache.maven.plugins</groupId>
      <artifactId>maven-compiler-plugin</artifactId>
      <configuration>
        <annotationProcessorPaths>
          <path>
            <groupId>io.github.joke.percolate</groupId>
            <artifactId>percolate</artifactId>
            <version>{percolate-version}</version>
          </path>
          <!-- optional: reactive (Flux/Mono) container support
          <path>
            <groupId>io.github.joke.percolate</groupId>
            <artifactId>reactor</artifactId>
            <version>{percolate-version}</version>
          </path>
          -->
        </annotationProcessorPaths>
      </configuration>
    </plugin>
  </plugins>
</build>
----

Percolate does not auto-map fields by name. Every field you want produced needs a `@Map` directive (or is
supplied by a target constructor argument that a directive feeds). A method with no `@Map` directives
produces nothing â€” see xref:map-annotation.adoc[The @Map annotation].
